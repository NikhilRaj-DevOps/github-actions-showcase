# Contributing

Thanks for your interest in improving this GitHub Actions showcase project.

## How to contribute

1. Fork the repository.
2. Create a branch for your change.
3. Make your update and keep the project simple and readable.
4. Run the local test suite before opening a pull request.
5. Open a pull request with a clear summary of the change.

## Local validation

```bash
python3 -m unittest discover -s tests -v
```

## Expectations

- Keep workflows readable and focused on real GitHub Actions examples.
- Prefer small, intentional changes.
- Document notable workflow behavior in the README when needed.
